# 性能优化
#### 如何做性能优化
```js
1. 运行时性能问题
  - 路由懒加载，动态引入组件，如import或者resolve组件
  - 使用keep-alive
  - v-if 代替 v-show
  - computed 和 watch 区分使用场景
  - v-for的key使用标志性参数，不要使用index，避免不必要的渲染
  - 使用生产环境的代码，减小不必要的代码
  - css提取到单独文件，因为css不会阻碍页面加载，js会
  - vuex的数据扁平化，避免层次太深
  - 合并请求，减少请求次数
  - 优化无限滚动的list，例如懒加载以及只渲染几十个DOM，滚出屏幕一定距离的都不渲染
  - 尽量减少引用类型数据的持久化（localstorage）
  - 事件的销毁，及时销毁不必要的事件
  - 第三方插件的按需引入

2. 加载时性能问题
  - 服务端渲染，减少首屏加载时间
  - 懒加载

3. 编译时优化
  - 开启 gzip 压缩
  - CDN 的使用
```
