4. 常见http状态码
  - 1xx: 指示信息
  - 2xx: 成功
    - 200：客户端请求成功
    - 206：客户端发送了一个带Range（指定范围）头的请求，服务器完成了它
  - 3xx: 重定向
    - 301：请求的页面已转移至新的url
    - 302：请求的页面已临时转移至新的url
    - 304：客户端有缓存的文件，服务端告诉客户端缓存文件可用
  - 4xx: 客户端错误
    - 400：客户端请求语法错误
    - 401：未授权
    - 403：禁止访问
    - 404：资源不存在
  - 5xx: 服务端错误
    - 500：服务器发生了不可预期的错误
    - 503：服务器过载或宕机，稍后可能恢复

5. 常见请求方式及区别
  - get：获取资源
  - post：传输资源
  - put：更新资源
  - delete：删除资源
  - head: 获得报文首部

6. 管线化
  - 原本：请求1 -> 响应1 -> 请求2 -> 响应2
  - 管线化：请求1 -> 请求2 -> 响应1 -> 响应2
  - 仅http1.1支持
  - 只有get和head支持，post受限制
  - 管线化不会影响响应回来的顺序

## 网络安全
1. CSRF
  - 基本概念
    - 跨站请求伪造
  - 攻击原理
    - 用户在A网址登陆过，本地存有A的cookie
    - 用户访问B网址的时候，B诱导用户访问A
    - A网址发现有cookie，则认为是正常访问
  - 防御措施
    - tooken验证
    - 隐藏令牌(例如放在head中)
    - Referer(页面来源)验证

2. XSS
  - 基本概念
    - 跨域脚本攻击
  - 攻击原理
    - 在提交区域里面加入html或script标签
  - 防御措施
    - 提交内容时校验内容

## 通信
1. 什么是同源/跨域
  - 同源：协议，域名，端口都相同，则同源，否则为跨域

2. 跨域有什么限制
  - cookie和浏览器的storage不能共享
  - DOM无法获得
  - ajax不能发送

3. 跨域的几种解决方法
  - jsonp，hash，domain，postMessage
  - websocket
  - cors
  - 反向代理

4. 前后端通信种类及各自优缺点
  - ajax
    - 同源下的通信方式
  - websocket
    - 不受同源策略限制
    - 长连接
  - cors
    - 不受同源策略限制
